# Outage Response Script

When your site breaks, do not panic-scroll. Follow this order.

## 1. Confirm the problem

- Open the site on your phone using cellular data.
- Open it in a private/incognito browser window.
- Test the exact customer path: view page → click button → submit/buy/book.

## 2. Identify the type of failure

- Whole site down
- One page down
- Checkout/payment broken
- Contact/lead form broken
- Site works but is very slow
- SSL/security warning
- Wrong content/pricing/hours shown

## 3. Immediate customer-safe action

Use the fastest temporary fix:

- Add a banner: “We’re fixing checkout. Email us at ___ for orders.”
- Update social bio/link if the main link is broken.
- Switch checkout/contact link to a backup form/payment link.
- Pause ads if paid traffic is being sent to a broken page.

## 4. Message template

“Quick heads up: our [checkout/contact/booking page] is having a temporary issue. We’re fixing it now. If you need help right away, use [backup email/link]. Thanks for your patience.”

## 5. After it is fixed

- Test from phone and desktop.
- Save what broke and how it was fixed.
- Add a monitor so the same issue is caught faster next time.
