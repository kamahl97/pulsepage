# Money Page Monitoring Checklist

Monitor the pages that directly affect revenue, not just the homepage.

## Monitor these first

- Homepage
- Product/service page
- Checkout page
- Booking page
- Contact page
- Lead form confirmation page
- Login/account page if customers need it
- API/status endpoint if your app depends on one

## What to watch

- Page is reachable
- SSL certificate is valid
- Response time is not unusually slow
- Checkout/contact path still works
- Important images/scripts are not broken
- Domain renewal date is known
- Hosting bill/payment method is current

## Suggested schedule

- Homepage: every 15-30 minutes
- Checkout/booking/contact: every 15 minutes
- SSL/domain review: monthly
- Full manual phone test: weekly

## Warning signs

- Site loads sometimes, fails sometimes
- Mobile view is broken
- Contact form says “sent” but no email arrives
- Checkout works in test mode but not live mode
- SSL warning appears
- Plugin/theme update changed layout
- Speed suddenly gets worse

## Simple rule

If a page can make or lose money, it deserves a monitor.
