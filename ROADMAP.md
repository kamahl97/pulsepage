# PulsePage Roadmap

PulsePage is moving from working prototype to testable product.

## Now: GitHub + Play Store readiness

- Keep GitHub repo clean and free of secrets/private runtime data.
- Maintain a clear MVP backlog.
- Resume Google Play Console submission.
- Run internal/closed testing with a small tester group.

## MVP v1.0

Goal: a simple Android/web uptime monitor people can understand in under 60 seconds.

Must have:
- Website monitor creation
- Server/port monitor creation
- Up/down/slow status display
- Response time history
- SSL expiration display
- Basic alert/outbox reporting
- Privacy policy and support pages
- Google Play closed testing build

Not required for v1.0:
- Full account system
- Payments/subscriptions
- Team accounts
- Native push notifications
- Enterprise features

## v1.1 polish

- Better onboarding copy
- Cleaner empty states
- More helpful error messages
- Public status page improvements
- Simple backup/export plan
- Better alert delivery options

## v2 monetization candidates

- Free tier: limited monitors/check interval/history
- Pro tier: more monitors, faster checks, longer history
- SMS alerts later, only after cost controls are clear
- Custom status page branding
- Custom domains

## Strategic direction

PulsePage should stay simple. The moat is not having the most features; it is helping small operators protect their money pages without learning DevOps.
