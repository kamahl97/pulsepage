# PulsePage Product Plan

## Target customer

Side hustlers, freelancers, local businesses, creators, and small e-commerce owners who have a website but do not have a technical ops team.

## Core promise

"Know your site is down before your customers do."

## Pain points

- Website downtime loses money and trust.
- Existing tools feel technical or expensive.
- Small businesses do not want to configure complex monitoring.
- Many owners only discover downtime after a customer complains.

## MVP features

1. User adds a URL.
2. System checks it every few minutes.
3. System records up/down, response time, status code, and error.
4. User sees monitor dashboard.
5. User gets an alert when status changes to down/recovered.
6. Public status page can show current state.

## Possible paid features later

- More monitors
- Faster check intervals
- SMS/phone alerts
- Branded status page
- Multiple team members
- Incident history/export
- Custom domains
- Keyword checks
- SSL expiration checks
- Domain expiration reminders

## First monetization idea

Free:
- 1 monitor
- 10-minute checks
- Email/Discord alert

Starter paid:
- 5 monitors
- 5-minute checks
- public status page
- $5/month

Pro:
- 20 monitors
- 1-minute checks
- SMS alerts/custom branding
- $15/month

## Positioning

Simple uptime monitoring for people who are busy running a business, not managing servers.
