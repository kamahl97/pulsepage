# Google Play Data Safety Draft

This is a working draft for the Play Console Data Safety form.

## Data collected

### Personal info
- Email address: only if account login is added.
- Purpose: account management, app functionality.
- Shared: no, except service providers required to operate the app.

### App activity / user-generated content
- Website URLs and server targets added by the user.
- Purpose: app functionality, uptime monitoring.
- Shared: no, except backend hosting/service providers.

### Device or other IDs
- Push notification token, if alerts are enabled.
- Purpose: app functionality, sending alerts.

### Diagnostics
- Crash logs and app diagnostics, if analytics/crash reporting is added.
- Purpose: app performance and reliability.

## Data not collected
- Location
- Contacts
- SMS/call logs
- Photos/videos/files
- Audio
- Calendar
- Health data

## Security practices
- Data encrypted in transit: yes, required before public launch.
- Users can request deletion: should be supported before public launch.
