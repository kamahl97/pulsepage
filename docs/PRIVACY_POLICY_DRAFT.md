# PulsePage Privacy Policy Draft

Last updated: [DATE]

PulsePage is a website and server uptime monitoring app. This draft explains what information PulsePage collects and why.

## Information we collect
PulsePage may collect the following information:

- Monitor targets you add, such as website URLs or server host/port combinations.
- Basic account information, such as email address, if account login is added.
- Device push notification token, if push alerts are enabled.
- App diagnostics, such as errors or crash logs, to help improve reliability.
- Subscription status, if paid plans are added through Google Play Billing.

## Information we do not collect
PulsePage does not need or collect:

- Contacts
- SMS or call logs
- Precise location
- Photos or files
- Camera access
- Microphone access
- Health data

## How we use information
We use collected information to:

- Check whether your monitor targets are online.
- Show uptime status and recent history.
- Send alerts when a monitor goes down or recovers.
- Manage subscription access if paid features are added.
- Improve app reliability and fix bugs.

## Data sharing
We do not sell personal data.

We may use trusted service providers for hosting, notifications, analytics, crash reporting, or payments. These providers are used only to operate PulsePage.

## Data security
We use reasonable safeguards to protect app data. Backend communication should use HTTPS before public launch.

## Data retention
Monitor history may be kept for a limited time depending on the user's plan. Users should be able to delete monitors they no longer want tracked.

## Children's privacy
PulsePage is not designed for children and does not target children.
